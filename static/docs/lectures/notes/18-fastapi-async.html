第18讲 笔记：FastAPI 与异步 Web 框架¶

学习目标¶

  • 能解释同步和异步编程在 Web 服务中的区别
  • 能说出 FastAPI 路由、中间件、依赖注入的基本用法
  • 能理解 WebSocket 在流式输出中的作用

核心知识点¶

  1. async/await 的核心价值：IO 等待时不阻塞线程，让单个线程能处理更多并发请求
  2. FastAPI 的路由层负责接收 HTTP 请求和 WebSocket 连接，不做业务逻辑
  3. WebSocket 连接可以持续双向通信，适合流式推送 token
  4. 同步生成器放在 asyncio.to_thread 中执行，避免阻塞事件循环
  5. CORS 中间件控制跨域访问，当前默认只允许本地 8000 端口

关键概念¶

术语 定义
async/await Python 异步编程语法，await 挂起当前协程让出控制权
事件循环 管理协程调度的核心机制，单线程轮转执行多个协程
WebSocket 全双工通信协议，一次握手后持续双向传输
依赖注入 FastAPI 的 Depends() 机制，在路由函数调用前自动执行依赖函数
CORS 跨域资源共享，控制哪些来源可以访问 API

# qa_core/api/chat.py — WebSocket 流式问答
@router.websocket("/api/stream")
async def websocket_endpoint(websocket: WebSocket):
    await websocket.accept()
    while True:
        raw_data = await websocket.receive_text()
        request_data = json.loads(raw_data)
        context = QueryServiceContext.from_ws_payload(request_data)

        stream = get_qa_service().stream_query(*context.service_args())
        while True:
            # 在线程中推进同步生成器，不阻塞事件循环
            has_event, event = await asyncio.to_thread(_next_stream_event, stream)
            if not has_event or event is None:
                break
            await websocket.send_json(event)
            if event.get("type") in {"end", "error"}:
                if event.get("type") == "end":
                    _schedule_summary_refresh(session_id)  # 后台异步刷新摘要
                break

# qa_core/api/chat.py — 安全推进同步生成器
def _next_stream_event(iterator):
    try:
        return True, next(iterator)
    except StopIteration:
        return False, None

练习题¶

  1. 模拟并发请求：用 asyncio.gather 同时发起 3 个 asyncio.to_thread 调用，验证它们是否真正并行执行。
  2. 阅读 WebSocket 路由：在 qa_core/api/chat.py 中找到 websocket_endpoint 函数，画出从接收消息到发送事件的数据流图。

常见面试题¶

Q1：为什么 WebSocket 流式输出要放在线程里？

QAService.stream_query() 是同步生成器，内部顺序执行 Milvus 检索、本地 rerank 和 LLM 调用。如果直接在 async 函数里调用 next()，会阻塞事件循环。用 asyncio.to_thread 把同步工作放到独立线程，事件循环继续处理其他连接。

Q2：/api/query 和 /api/stream 的分工是什么？

/api/query 是轻量预检，只处理问候/越界/人工客服等无需检索的问题，复杂问题返回 stream_required 让前端走 /api/stream。这样避免同一问题触发两套生成逻辑，防止重复检索和历史重复写入。
