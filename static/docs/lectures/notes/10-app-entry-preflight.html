第10讲 笔记：应用入口与环境前置校验¶

学习目标¶

  • 能说清 app.py 的四项职责
  • 能解释为什么项目不允许降级启动
  • 能列出启动时校验的全部 6 项前置条件

核心知识点¶

  1. app.py 只做四件事：创建 FastAPI 应用、配置 CORS 和静态资源、启动时校验和预热、注册路由
  2. Preflight Check 是航空术语「起飞前检查单」在软件工程中的应用：缺了核心依赖就拒绝启动，不做运行时降级
  3. 启动校验链覆盖 6 项：LLM API Key 非占位符、LLM 联通性、MySQL 可连接、Milvus TCP 可达、模型路径存在、场景配置目录有效
  4. 预热 warmup_retrieval_stack() 在启动时加载全部场景的 Embedding 和 Reranker 模型，避免第一个用户等待 30 秒
  5. 路由拆分为四个模块：pages（页面）、chat（问答）、admin（管理）、kb_versions（版本）

关键概念¶

术语 定义
Preflight Check 服务启动前校验所有核心依赖是否可用
Fail-Fast 遇到不可恢复的错误立即失败，不隐藏问题
技术降级 缺少某个组件时自动切换到低配方案（本项目禁止）
预热 启动时主动加载模型和建立连接，避免首次请求时的冷启动延迟
lru_cache 进程级单例缓存，避免每次请求都重新创建服务对象

# qa_core/llm/client.py — LLM 客户端工厂 + 连通性验证
@lru_cache(maxsize=2)
def get_chat_model(streaming: bool = False) -> ChatOpenAI:
    settings = get_settings()
    return ChatOpenAI(
        model=settings.llm_model,
        api_key=settings.llm_api_key,
        base_url=settings.llm_base_url,
        temperature=settings.llm_temperature,
        timeout=settings.llm_timeout,
        streaming=streaming,
    )

def validate_llm_connectivity() -> None:
    """真实调用一次 LLM，确认 Key、URL、模型名都可用。"""
    response = get_chat_model(streaming=False).invoke(
        [HumanMessage(content="回复 OK")]
    )
    if not str(getattr(response, "content", "") or "").strip():
        raise RuntimeError("LLM 服务返回空内容")

# qa_core/config/preflight.py — TCP 连通性检查
def _require_tcp(name, host, port, timeout=3.0):
    sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    sock.settimeout(timeout)
    result = sock.connect_ex((host, port))
    sock.close()
    if result != 0:
        raise RuntimeError(f"{name} ({host}:{port}) 不可连接，请检查服务是否已启动")

练习题¶

  1. 追踪启动流程：在 app.py 和 validate_runtime_environment() 中加临时日志，重启服务，观察各项校验的执行顺序和耗时。
  2. 模拟校验失败：把 .env 中的 MILVUS_URI 改为无效地址，重启服务，确认启动是否立即失败并给出明确错误信息。

常见面试题¶

Q1：为什么项目不允许降级启动？

这是一个教学项目，环境是可控的。降级启动（如缺 Milvus 时换 pgvector）会让学生在半残环境里排查奇怪问题。严格的 fail-fast 策略保证了问题第一时间暴露，且保证走的是唯一主链路。

Q2：启动预热做了什么？为什么不放在首次请求时懒加载？

预热加载 Embedding 模型（2.2GB BGE-M3）和 Reranker 模型（1.3GB），约需 10-30 秒。如果放在首次请求时懒加载，第一个用户会等待 30 秒才能看到答案，体验很差。
