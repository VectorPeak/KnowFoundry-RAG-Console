第4讲 笔记：检索策略与动态计划¶

学习目标¶

  • 能解释为什么不同问题需要不同的检索参数
  • 能说出 RetrievalPlan 的关键字段及其作用
  • 能理解费用/合规类问题的特殊保护策略

核心知识点¶

  1. 不用所有问题一套参数：FAQ 用低阈值勇敢直出，知识咨询扩大文档候选，追问提高阈值防误命中，短句显歧义需保护
  2. RetrievalPlan 是一个 frozen=True 的 dataclass，一旦构建不可修改，表示一次确定性的检索决策
  3. 问答类别保护是检索策略中最重要的安全机制：pricing 阈值 ≥0.84、compliance 阈值 ≥0.86（最高），宁可多召回也不轻信一条 FAQ
  4. 表格类问题强制 faq_direct_exact_only=True：只允许标准问题精确匹配，禁止相似 FAQ 直出
  5. prefer_table 标志还会影响 select_context_docs 的排序逻辑：同等分数下表格行优先于普通文本段落

关键概念¶

术语 定义
RetrievalPlan 封装单次检索所有参数的不可变对象
Dynamic Threshold 按问题类型动态调整的 FAQ 直出阈值
Short Query Guard 短问题保护：收缩文档候选数，提高直出阈值
Pricing Guard 费用类特殊保护：阈值 ≥0.84，扩大文档召回

# qa_core/retrieval/strategy.py — 核心策略分支
def build_retrieval_plan(query: str, intent: IntentResult) -> RetrievalPlan:
    settings = get_settings()
    question_category = infer_question_category(query.strip())
    prefer_table = is_table_query(query.strip())
    is_short = len(query.strip()) <= settings.short_query_max_chars

    if intent.intent == "FAQ_QUERY":
        doc_top_k = max(8, settings.doc_top_k // 2)
        direct_threshold = max(0.62, settings.faq_direct_score_threshold - 0.08)
    elif intent.intent == "KNOWLEDGE_QUERY":
        doc_top_k = max(settings.doc_top_k, settings.doc_complex_query_top_k)
        final_context_top_n = max(settings.final_context_top_n, 5)
    elif intent.intent == "FOLLOW_UP":
        direct_threshold = max(settings.faq_direct_score_threshold, 0.78)

    # 问题类别保护（核心安全机制）
    if question_category == "pricing":
        direct_threshold = max(direct_threshold, 0.84)
    elif question_category == "compliance":
        direct_threshold = max(direct_threshold, 0.86)  # 最高阈值

    # 表格类：禁止相似 FAQ 直出
    if prefer_table and run_doc:
        faq_direct_exact_only = True

练习题¶

  1. 对比不同意图的检索计划：分别以一句 FAQ、一句知识咨询、一句追问作为输入，打印出 build_retrieval_plan() 返回的完整参数，对比差异。
  2. 模拟费用类保护：在 .env 中临时降低 FAQ_DIRECT_SCORE_THRESHOLD 到 0.5，然后提问「退款多久到账」，观察 pricing guard 是否将阈值拉回到 0.84 以上。

常见面试题¶

Q1：为什么 FAQ 直出不能用一个固定阈值？

不同类型问题的「误直出」代价不同。FAQ 类问题误直出最多答偏，费用/合规类问题误直出可能承诺错误金额或给出违法建议。所以用动态阈值：FAQ 类可以低至 0.62，合规类必须 ≥0.86。

Q2：表格问题为什么强制 faq_direct_exact_only=True？

用户问表格问题时通常是想查某个字段的具体值，泛化 FAQ 只能答出「需要哪些材料」而非具体状态。只有标准问题完全匹配时才敢直出，相似命中必须走文档 RAG。
