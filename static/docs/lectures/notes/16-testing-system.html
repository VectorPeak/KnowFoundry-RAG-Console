第16讲 笔记：测试与接口验收¶

学习目标¶

  • 能画出 RAG 测试金字塔的三层结构
  • 能说出纯逻辑测试为什么绕过 HTTP 直接调用核心函数
  • 能列出项目至少 5 个测试用例覆盖的维度

核心知识点¶

  1. 测试金字塔三层：纯逻辑单元测试（无外部依赖，快）→ 验收逻辑测试（模拟报告数据）→ E2E 验收测试（完整环境，慢）
  2. 纯逻辑测试绕过 HTTP 是因为 RAG 测试有 4 个挑战：外部依赖重、答案不确定、慢、消耗 LLM token
  3. 当前测试覆盖 104 个 pytest 用例，维度包括：意图识别 6 种类型、场景/边界检测、source 推断、检索过滤、上下文构建、Prompt 选择、FAQ 直出判断、引用增强、入库质量检查、Bad Case、表格/OCR 风险和质量报告
  4. 评测集不属于单元测试，而是回归验证：用 evaluate_core_chain.py 定期跑全量评测集
  5. 接口验收由 api_e2e_smoke.py 和 acceptance_smoke.py 分别覆盖管理接口、页面和 WebSocket 流式事件

关键概念¶

术语 定义
纯逻辑测试 不依赖 Milvus/MySQL/LLM，直接调用核心函数验证逻辑
测试金字塔 底层大量快测试 + 中层适量半真实测试 + 顶层少量慢全链路测试
验收测试 用模拟的验收报告数据验证 pass/fail 判定逻辑
质量检查 按主题执行项目守护、评测、性能和接口验收，任一项失败都需要先修复

# tests/test_intent_and_scenarios.py — 纯逻辑测试（无外部依赖）
class IntentClassifierTests(unittest.TestCase):
    def test_business_knowledge_uses_knowledge_intent(self):
        scenario = get_scenario_registry().resolve("enterprise_knowledge")
        result = classify_intent("新人入职流程怎么走", [], scenario)
        self.assertEqual(result.intent, "KNOWLEDGE_QUERY")
        self.assertEqual(result.suggested_source, "hr")

    def test_short_direct_faq_prefers_faq_intent(self):
        scenario = get_scenario_registry().resolve("enterprise_knowledge")
        result = classify_intent("员工报销需要准备哪些材料？", [], scenario)
        self.assertEqual(result.intent, "FAQ_QUERY")
        self.assertEqual(result.reason, "source_question_shape_rule")

class ScenarioRegistryTests(unittest.TestCase):
    def test_scenario_boundary_detects_cross_scene(self):
        scenario = get_scenario_registry().resolve("enterprise_knowledge")
        decision = detect_scenario_boundary(
            "安全技术交底只有口头说明可以吗？", scenario)
        self.assertTrue(decision.crossed)
        self.assertEqual(decision.matched_scenario_id, "engineering_project_qa")

    def test_8_scenarios_all_registered(self):
        registry = get_scenario_registry()
        ids = {s.scenario_id for s in registry.list_scenarios()}
        self.assertEqual(len(ids), 8)
        self.assertIn("enterprise_knowledge", ids)
        self.assertIn("saas_support", ids)
        self.assertIn("cross_border_risk", ids)
        self.assertIn("insurance_claims", ids)

练习题¶

  1. 跑一遍全部测试：执行 python -m pytest tests -q，确认 104 个 pytest 用例全部通过，阅读每条测试的断言理解它在验证什么。
  2. 新增一个测试用例：仿照 test_enterprise_source_patterns_are_used_for_source_inference，为 engineering_project_qa 场景新增一条 source 推断测试。

常见面试题¶

Q1：RAG 系统为什么不能只用 E2E 测试？

RAG 答案是不确定的自然语言。E2E 测试需要完整环境（Milvus+MySQL+LLM），一次 2-5 秒，跑 100 条消耗大量时间和 token。纯逻辑测试绕过 HTTP 直接测意图识别、检索过滤等确定性逻辑，快速、低成本、可精确断言。

Q2：你们的测试能在 CI 里跑吗？

纯逻辑测试和静态验收检查可以在 CI 里跑，不需要外部服务。E2E 验收测试和RAG 回归验收需要完整环境，适合在发布前的本地或预发布环境中执行。
