第3讲 笔记：意图识别系统¶

学习目标¶

  • 能说出 6 种意图类型的含义和处理方式
  • 能解释「规则优先 + LLM 补充」的分层策略
  • 能理解 source 自动推断如何辅助检索过滤

核心知识点¶

  1. 6 种意图：GREETING（问候直答）、FOLLOW_UP（追问改写）、KNOWLEDGE_QUERY（文档 RAG）、FAQ_QUERY（FAQ 优先）、HUMAN_SERVICE（客服直答）、OUT_OF_SCOPE（拒答）
  2. 决策顺序决定准确性：问候→越界→人工客服→追问→强规则→LLM 补充，规则拦截住高频确定场景后再让 LLM 处理模糊问题
  3. 强规则覆盖六类场景：强 FAQ 关键词 + source 已推断 + 问法像标准问答 → 直接定 FAQ_QUERY，跳过 LLM
  4. source 自动推断只使用场景 TOML 的 source_patterns，没有场景配置时返回空，不在代码中保留业务分类硬编码
  5. with_structured_output 约束 LLM 只返回 Pydantic 定义的枚举字段，suggested_source 不在 valid_sources 白名单中时会被丢弃

关键概念¶

术语 定义
IntentResult 标准化意图结果，包含 intent / confidence / reason / requires_rewrite / suggested_source
规则优先 高频确定场景用正则规则（快、稳），模糊场景才调 LLM（灵活）
Source 推断 根据问题关键词匹配场景 TOML 的 source_patterns，自动推断业务分类
direct_answer 非空时跳过检索直接返回，用于问候/越界/人工客服

# qa_core/intent/classifier.py — 六步判断顺序
def classify_intent(query, history, scenario=None):
    normalized = query.strip().lower()

    # 1. 问候 → 直接答案
    for pattern, answer in zip(GREETING_PATTERNS, greeting_answers):
        if re.match(pattern, normalized, re.IGNORECASE):
            return IntentResult(intent="GREETING", direct_answer=answer, confidence=1.0)

    # 2. 越界 → 拒答
    if OFF_TOPIC_HINTS.search(normalized):
        return IntentResult(intent="OUT_OF_SCOPE", direct_answer=f"超出了{business_domain}范围",
                           confidence=0.95, reason="safety_rule")

    # 3. 人工客服短句 → 直接返回电话
    if HUMAN_SERVICE_HINTS.search(normalized) and len(normalized) <= 18:
        return IntentResult(intent="HUMAN_SERVICE",
                           direct_answer=f"联系方式：{support_contact}", confidence=0.9)

    # 4. 有历史 + 短句/代词 → 追问
    if history and (FOLLOW_UP_HINTS.search(query.strip()) or len(query.strip()) <= 8):
        return IntentResult(intent="FOLLOW_UP", confidence=0.8, requires_rewrite=True)

    # 5. 强规则 → FAQ_QUERY / KNOWLEDGE_QUERY
    strong = _strong_rule_domain_intent(query, suggested_source)
    if strong is not None:
        return strong

    # 6. 仍不确定 → LLM 结构化输出
    return _classify_with_llm(query, history, suggested_source, scenario)

def infer_source(query, scenario=None):
    if scenario is None:
        return None
    best_source, _ = score_source_matches(query, scenario)  # 只使用当前场景 TOML 正则
    return best_source

练习题¶

  1. 追踪意图判断路径：在 classify_intent() 中加临时日志，分别输入一句问候、一句 FAQ 和一句复杂问题，观察走了哪条判断路径和原因。
  2. 新增 source 关键词：在某个场景的 scenario.toml 中新增一个 source 和对应的 source_patterns 正则，重启后提问包含该关键词的问题，观察 suggested_source 是否正确。

常见面试题¶

Q1：为什么不全部用 LLM 做意图识别？

问候、越界、人工客服等高频场景确定性强，用 LLM 反而增加延迟和成本。规则覆盖常见请求（~80%），LLM 只处理模糊问题（~20%），成本和稳定性都更优。

Q2：如果 LLM 返回的 suggested_source 不在 valid_sources 白名单中怎么办？

代码会丢弃模型输出的 suggested_source，回退到规则推断的 suggested_source。无效的分类名不会进入 Milvus 过滤表达式，避免查询失败。
