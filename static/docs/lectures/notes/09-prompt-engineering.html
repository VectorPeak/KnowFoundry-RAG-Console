第9讲 笔记：Prompt 工程与 Profile 系统¶

学习目标¶

  • 能列出 8 种 Prompt Profile 及其适用场景
  • 能解释为什么用确定性选择而非让 LLM 自己判断模板
  • 能说出 System Prompt 编写的三个核心原则

核心知识点¶

  1. 8 种 Prompt Profile：faq_answer / knowledge_answer / follow_up / pricing_guard / compliance_guard / troubleshooting_steps / source_bound_summary / default_answer
  2. Prompt Profile 选择逻辑：先按 question_category 找专用模板 → 再按 intent 找通用模板 → 都不命中用 default
  3. 三个核心原则：身份（你是谁）→ 边界（你能做什么/不能做什么）→ 约束（回答格式和引用要求）
  4. 确定性选择优于 LLM 判断：代码规则可测试、可回滚、可审计，让 LLM 再选一次模板会引入不稳定性
  5. 场景变量注入：{assistant_name} {business_domain} {support_contact} 在运行时从 scenario.toml 填充

关键概念¶

术语 定义
Prompt Profile 一组预设的 System Prompt + User Prompt 模板，针对特定问题类型
System Prompt 定义 AI 身份和行为边界的指令，对整个对话生效
Pricing Guard 费用类专用模板，要求区分「已确认」和「未确认」信息
Compliance Guard 合规类专用模板，要求基于条款回答并标注依据

# qa_core/prompts/selector.py — 确定性选择
def build_answer_prompt_profile(intent, scenario=None, query=None):
    question_category = infer_question_category(query or "")
    # 优先按类别找专用模板（如 pricing_guard），再按意图找通用模板
    profile = (CATEGORY_PROMPT_PROFILES.get(question_category)
               or PROMPT_PROFILES.get(intent, DEFAULT_PROMPT_PROFILE))
    context = _scenario_prompt_context(scenario)
    return PromptProfile(
        name=profile.name,
        system_template=profile.system_template.format(**context),  # 注入场景变量
        user_template=profile.user_template,
        reason=profile.reason,
    )

练习题¶

  1. 对比不同 Profile 的回答：分别提问一句费用类问题（「退款多久到账」）和一句知识咨询（「入职流程有哪些步骤」），通过 trace 查询确认使用了正确的 Prompt Profile。
  2. 阅读 summary Profile：在 profiles.py 中找到 source_bound_summary，说明它为什么属于“按问题类别优先”的模板，而不是普通知识问答模板。

常见面试题¶

Q1：为什么不让 LLM 自己判断用哪个 Prompt 模板？

LLM 的选择不可靠且不可复现。同一个问题两次可能选出不同模板，回答口径不一致。代码规则的确定性选择可测试、可回归、可审计。

Q2：费用类和合规类的 Prompt 有什么关键区别？

费用类模板强调「区分已确认和未确认信息」，防止模型对退款金额、到账时间等做出未核实的承诺。合规类模板要求「基于条款回答并标注具体条款依据」，且阈值更高（0.86），因为合规判断涉及法律风险。
