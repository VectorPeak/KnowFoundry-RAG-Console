第8讲 笔记：RAG Pipeline 主流程¶

学习目标¶

  • 能画出七阶段 RAG Pipeline 的完整流程图
  • 能解释 FAQ 快速路径和 FAQ 标准直出的区别及触发条件
  • 能说出上下文构建中的筛选/去重/截断/格式化四步操作
  • 能描述流式事件协议的五种事件类型

核心知识点¶

  1. Pipeline 分七阶段：FAQ 快速路径→意图识别→FAQ 检索（可能直出）→文档检索→上下文构建→LLM 生成→引用增强
  2. FAQ 快速路径在意图识别之前执行：只匹配「短、完整、像标准问答」的问题，且只允许精确匹配
  3. Pipeline 有三个提前出口：FAQ 快速路径命中、直接答案（问候/越界）、FAQ 高分直出
  4. 上下文构建顺序：FAQ 补充 → 分数过滤 → 去重 → 优先表格行 → 长度截断 → 格式化编号
  5. 五种 WebSocket 事件：start / status / token / end / error，前端逐个消费

关键概念¶

术语 定义
Fast Path 在意图识别前用原问题直接搜 FAQ，精确匹配则直出
Parent Content metadata 中保存的父块全文，检索命中子块时展开为完整上下文
Insufficient Context 过滤后无可用上下文，不调 LLM，直接返回固定口径
Citation Enforcement 检查答案是否含 [N] 引用编号，没有则追加参考来源

# qa_core/pipeline/rag.py — 七阶段主流程
def stream_query(history, validate_source, query, source_filter, session_id, ...):
    context = create_query_context(...)
    yield build_query_start_event(context)

    # 第零阶段：FAQ 快速路径（意图识别前，精确匹配）
    fast_answer = try_fast_faq_direct_answer(context)
    if fast_answer:
        yield finish_success(context, answer=fast_answer); return

    # 第一阶段：意图识别 + 检索准备
    prepared = prepare_retrieval(context)
    if prepared.intent.direct_answer:  # 问候/越界直接结束
        yield finish_success(context, answer=prepared.intent.direct_answer); return

    # 第二阶段：FAQ 检索（可能直出）
    faq_result = search_faq(context, prepared)
    direct_answer = get_faq_direct_answer(context, prepared, faq_result)
    if direct_answer:
        yield finish_success(context, answer=direct_answer); return

    # 第三阶段：文档检索 → 上下文构建 → LLM 流式生成
    doc_result = search_doc(context, prepared)
    answer_prepared = prepare_answer(context, prepared, faq_result, doc_result)
    for chunk in stream_llm_answer(system_prompt, user_prompt):
        token = str(getattr(chunk, "content", "") or "")
        if token:
            yield build_token_event(token, context.session_id)

    # 引用增强 + 保存历史
    answer = enforce_answer_citations(context.answer, answer_prepared.context_docs)
    history.add_turn(context.session_id, query, answer)
    yield finish_success(context, answer=answer)

# qa_core/pipeline/steps.py — FAQ 快速路径的精确匹配
def _exact_faq_answer(query, faq_result):
    """只允许标准问题完全一致时直出，不按相似分数直出"""
    for hit in faq_result.hits:
        answer = direct_faq_answer(query, hit.document, hit.score, threshold=float("inf"))
        if answer:
            return answer, faq_result
    return None, faq_result

练习题¶

  1. 追踪完整链路：用一个简短 FAQ 问题通过 WebSocket 提问，在浏览器 Console 中打印收到的每个事件，画出 start→status→token→end 的时序。
  2. 验证 Parent 展开：通过 debug_retrieval 查一个文档类问题，检查返回的 parent_content 和 page_content 的差异。

常见面试题¶

Q1：FAQ 快速路径和后续 FAQ 检索有什么区别？

快速路径在意图识别前执行，只匹配精确相同的问题，缩短标准 FAQ 的首 token。后续 FAQ 检索在意图识别后，由 qa_core/pipeline/retrieval_steps.py 执行，允许按相似分数直出（动态阈值），覆盖面更广。两者互补，fast path 结果会复用给后续 search_faq。

Q2：信息不足时为什么不调 LLM 而直接返回固定提示？

没有可用上下文时，让 LLM 自由发挥最容易产生幻觉。直接返回固定口径「信息不足，请联系人工」，比让模型在没有资料的情况下编造答案安全得多。
